## About the Project

Prept is a full-stack SaaS platform that solves two problems for software engineers:

**1. Mock Interviews** — Connect with senior engineers from top companies (Google, Meta, Amazon, etc.) for 1:1 video mock interviews. Get AI-powered feedback reports generated automatically from call transcripts after every session.

**2. Project Management** — Create workspaces, invite team members, manage projects with a Kanban board, and track tasks — all tailored for interview prep and job application cycles.

Built with a credit-based subscription model: interviewees buy credits to book sessions, interviewers earn credits and withdraw earnings.

---

## Features

### Interview Platform

- **Interviewer Profiles** — Browse senior engineers by specialty, company, and availability
- **Slot-based Scheduling** — Interviewers set availability windows; interviewees pick 45-minute slots
- **HD Video Calls** — Powered by Stream with screen sharing, recording, and transcription
- **Persistent In-Call Chat** — Real-time messaging during the session
- **AI Question Generator** — Live AI co-pilot generating role-specific questions (DSA, System Design, Behavioral, etc.) during the call
- **AI Feedback Reports** — Post-interview analysis generated by Gemini from call transcripts — technical assessment, communication, problem-solving, strengths, improvements, and hire recommendation
- **Credit System** — Subscription-based credits; interviewers earn and withdraw via PayPal, Bank Transfer, or UPI
- **Rate Limiting** — Arcjet bot protection on all sensitive API routes

### Project Management

- **Workspaces** — Create and manage multiple workspaces per account
- **Team Invitations** — Invite collaborators via email with tokenized invitation links (built from scratch — not using Clerk's built-in organization feature)
- **Projects** — Create categorized projects (Web, Mobile, ML/AI, Design, etc.) with timeline tracking
- **Kanban Board** — Drag-and-drop task management with TODO / IN PROGRESS / IN REVIEW / DONE columns
- **Task Management** — Rich tasks with Markdown descriptions, priority levels, assignees, and due dates
- **Filtering** — Filter tasks by assignee, priority, and search term

### Platform

- **Authentication** — Clerk-powered auth with social login support
- **Subscription Plans** — Free, Starter, and Pro tiers via Clerk billing
- **Transactional Emails** — Invitation emails and withdrawal request notifications
- **Responsive** — Desktop-first design with mobile fallback cards
- **Dark Mode** — Dark theme throughout

## Getting Started

### Prerequisites

Make sure you have the following installed:

- **Node.js** >= 18.17.0
- **npm** >= 9.0.0 or **pnpm** >= 8.0.0
- **Git**

You will also need accounts for the following services (all have free tiers):

- [Clerk](https://clerk.com) — Auth
- [Supabase](https://supabase.com) — Database
- [Stream](https://getstream.io) — Video and Chat
- [Google AI Studio](https://aistudio.google.com) — Gemini API
- [Arcjet](https://arcjet.com) — Security
- A Gmail account — Email sending

---

### Installation

**1. Clone the repository**

```bash
git clone https://github.com/yourusername/prept.git
cd prept
```

**2. Install dependencies**

```bash
npm install
```

**3. Set up environment variables**

```bash
cp .env.example .env.local
```

Fill in all values in `.env.local` — see the [Environment Variables](#environment-variables) section below.

**4. Generate Prisma client**

```bash
npx prisma generate
```

**5. Run database migrations**

```bash
npx prisma migrate deploy
```

**6. Start the development server**

```bash
npm run dev
```

Open [http://localhost:3000](http://localhost:3000) in your browser.

## Deployment on Vercel

### 1. Push to GitHub

```bash
git add .
git commit -m "initial commit"
git push origin main
```

### 2. Import to Vercel

1. Go to [vercel.com](https://vercel.com) and click **Add New Project**
2. Import your GitHub repository
3. Framework preset will auto-detect **Next.js**

### 3. Add Environment Variables

In Vercel project settings → **Environment Variables**, add every variable from your `.env.local` file.

Make sure to update:
- `NEXT_PUBLIC_APP_URL` → your production Vercel URL (e.g., `https://prept.vercel.app`)

### 4. Add Build Command Override

In Vercel project settings → **General → Build and Development Settings**:

- **Build Command:** `npx prisma generate && next build`

Or ensure your `package.json` has:

```json
{
  "scripts": {
    "build": "next build",
    "postinstall": "prisma generate"
  }
}
```

### 5. Deploy

Click **Deploy**. Vercel will build and deploy your app automatically.

### 6. Post-Deploy Steps

After your first successful deployment:

1. Update `NEXT_PUBLIC_APP_URL` in Vercel env vars to your actual domain
2. Update the webhook URL in your **Stream dashboard** to your production domain
3. Update the webhook URL in your **Clerk dashboard** to your production domain
4. Redeploy once after updating env vars

## Key Workflows

### Interview Booking Flow

```
Interviewee browses /explore
    → Views interviewer profile at /interviewers/[id]
    → Selects a date and available time slot
    → Confirms booking (credits deducted atomically)
    → Stream video call room created
    → Redirected to /appointments
    → Joins call at /call/[callId]
    → After call ends, Stream sends webhook
    → Transcript downloaded and sent to Gemini
    → AI feedback saved to database
    → Booking marked as COMPLETED
    → Interviewer credit balance incremented
```

### Withdrawal Flow

```
Interviewer requests withdrawal from /dashboard → Earnings tab
    → Credits deducted from balance atomically
    → Payout record created with PROCESSING status
    → Admin receives email with review link
    → Admin visits /payout/[id] and enters password
    → Payout marked as PROCESSED
```

### Workspace Invitation Flow

```
Admin opens workspace → Manage → Invitation tab
    → Enters email addresses
    → Invitation records created in DB with unique tokens
    → Emails sent with accept/reject links
    → Recipient clicks link → /invite?token=xxx
    → If accepted and signed in → added as workspace member
    → If not signed in → prompted to sign in first
```

## Known Issues and Roadmap

### Known Issues

- Webhook-based AI feedback requires a public URL — does not work on localhost without ngrok
- Partial withdrawal is not supported — only full balance withdrawal
- Workspace listing only shows workspaces where the user is admin, not member-only workspaces

### Roadmap

- [ ] Partial credit withdrawal
- [ ] Interviewer rating system (post-session ratings)
- [ ] Email notifications for upcoming sessions
- [ ] Calendar integration (Google Calendar sync)
- [ ] Mobile responsive video call UI
- [ ] Workspace analytics dashboard
- [ ] Public interviewer profiles with review history
- [ ] Recurring availability slots
